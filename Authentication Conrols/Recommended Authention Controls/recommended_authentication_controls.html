Security Control:

AC-11: Session Lock

Identify, document, and enforce security controls on which user actions may be performed on a system without passing the matched-pair challenge response—for example, contacting technical support or making emergency calls.

Security Control:

AC-14: Permitted Actions Without Identification or Authentication

Develop matched-pair authentication processes that are resistant to replay attacks by establishing large sets of one-time challenge response authenticators.

Security Control:

IA-2: Identification and Authentication (Organizational Users) - (8) Network Access to Privileged Accounts—Replay Resistant

Capture information that uniquely identifies user devices requesting authentication to gain intelligence on unidentified adversaries who fail the matched-pair challenge response.

Security Control:

IA-3: Device Identification and Authentication - (4) Device Attestation

Require in-person matched-pair input to mitigate compromise of the challenge response identification system.
